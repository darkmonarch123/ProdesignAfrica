# Prodesign Africa — Build Slice 1: Core CAD Editor + Backend Skeleton

This is the first working slice of the full spec in `prodesign_v6_1.md`. It's a real,
runnable full-stack app — not a mockup — scoped to the piece the spec itself calls
the heart of the product: a working 2D CAD editor with 3D extrusion, backed by a
real auth + persistence API.

## What's working right now

**Frontend** (`/frontend`, React + TypeScript + Vite + Tailwind + Three.js)
- Full visual redesign matching the provided mockups: landing page, split-panel
  login/signup, dashboard with sidebar + stats + project grid, and editor chrome
  (dark topbar, vertical icon toolbar, properties panel, dark status bar) — all
  responsive (mobile nav collapses to a menu, dashboard sidebar becomes a
  horizontal pill scroller, editor props panel hides below `lg`).
- 2D CAD canvas: draw walls (click-click, chained, angle-snapped to 15°, endpoint
  snapping, grid snapping), draw multi-point rooms (click to add points, click near
  start to close), place doors/windows (auto-snaps to nearest wall, clamps to stay
  within wall bounds), select/delete elements, pan (middle-mouse or Pan tool), zoom
  (scroll wheel or toolbar buttons).
- Full undo/redo (Ctrl+Z / Ctrl+Y), keyboard delete, Escape to cancel in-progress draw.
- 3D view: toggles from the same canvas state — extrudes walls to boxes with real
  height/thickness, cuts door/window openings out of the wall run, floors rendered
  from room polygons, orbit camera, shadows, gizmo.
- Auth pages (login/signup — real API calls), dashboard (list/create projects),
  editor page wired to load/autosave snapshots (debounced, 1.5s) against the backend.
- **Compliance panel**: live setback/coverage/height/ROW checks against a real
  backend rule engine, refreshed automatically after every autosave. A "Compliance"
  button in the editor topbar opens plot-settings (plot width/depth + ruleset picker).
- **AI room planner**: describe a brief (building type, bedrooms, free-text
  description) and get a starter layout generated by Claude, loaded straight into
  the editor. Reachable via the **AI Layout** button in the editor topbar. Warns
  before overwriting an existing drawing, and the replacement is a real undo step
  (not a hard reset) so you can Ctrl+Z back to what you had.
- **Live collaboration**: open the same project in two browser tabs (or two
  people) and you'll see each other's cursor moving in real time, an avatar
  stack in the topbar showing who's currently active, and each other's edits
  appearing live. **Important limitation, stated plainly**: this is a
  last-write-wins broadcast, not a real merge (no CRDT/operational transform)
  — if two people draw at the exact same moment, whichever edit arrives last
  simply overwrites the other locally. It's solid for "watch a colleague
  sketch" or one-person-at-a-time handoff; it is not safe for truly
  simultaneous concurrent editing. The autosave snapshot (the actual saved
  data) is unaffected by this — a lost live-broadcast frame doesn't mean a
  lost save, as long as the person who made the edit is still connected long
  enough for their own autosave to fire.
- **BOQ export**: the **Export** button in the editor topbar opens a live Bill
  of Quantities preview, currency-switchable (NGN/GHS/KES/ZAR), with real Excel
  and PDF downloads styled as a categorized quantity sheet (header row, grouped
  categories, rate/quantity/amount columns, totals row) — computed from the
  actual drawn geometry, not placeholder numbers. **Scoped honestly**: it prices
  blockwork, doors, windows, and floor tiling, because that's what the drawing
  schema currently models. It does not include foundation, roofing, electrical,
  or plumbing quantities — those aren't represented in the canvas yet, so rather
  than fabricate numbers for them, the export says so explicitly in a Notes
  section on both the preview and the downloaded files.
- **AI room planner works with zero cost/setup now, too**: if no
  `ANTHROPIC_API_KEY` is configured, the AI Layout feature no longer just
  fails — it falls back to the same procedural generator behind the 1,088
  template catalog, producing a real layout from your bedroom count and plot
  size instantly, no API key or payment required. Stated honestly in the UI
  before you generate and again after: this fallback can't read the wording
  of a free-text brief the way real AI can, only the structured bedroom
  count/plot size. Set `ANTHROPIC_API_KEY` any time to get full free-text AI
  generation instead — nothing else about the feature changes.
- **WebSocket access control**: previously, `CollabController`'s @MessageMapping
  methods checked project access on every SEND (join/cursor/state), but STOMP
  SUBSCRIBE itself was never checked — any authenticated user who knew a
  project's UUID could subscribe directly to its `/topic/projects/{id}/state`
  channel and silently observe another team's live edits without ever calling
  join. `WebSocketAuthorizationInterceptor` closes this: every SUBSCRIBE to a
  project-scoped channel now runs through the same `ProjectService.requireAccess`
  check as everything else in the app, rejecting the subscription outright
  (not silently dropping it — the client gets a real STOMP ERROR frame) if the
  user has no access to that project.
- **Template realism upgrade**: after reviewing real Nigerian/Kenyan floor
  plan references, the generator now includes an Entrance Porch (full-width
  shallow front strip), a genuinely separate Dining room next to Living
  rather than one combined space, a Store/pantry next to the kitchen on
  3+ bedroom homes, and a private en-suite next to the master bedroom on
  4+ bedroom homes (in addition to the shared family bathroom) — all
  patterns pulled directly from the reference plans reviewed. Re-verified
  the same way as before: ported the updated coordinate math to Python and
  ran the full 1,088-combination sweep — 0 skipped, 0 errors, and every
  single generated template now has a split Living/Dining. **Still not
  attempted**: the angled/partial-width porches and corridor-with-side-doors
  layouts visible in some reference plans — those need a polygon-based
  generator rather than this rectangle-grid one, flagged as real follow-up
  work rather than risked under time pressure across a 1000+ template catalog.
- **Template marketplace**: browse 1,088 generated starter floor plans at
  `/templates` (linked from the dashboard sidebar), filterable by bedroom
  count, with a real schematic thumbnail per template. Pick one, name your
  project, and it clones straight into a new project with the layout already
  drawn. **How "1,088" is real, not padding**: these are procedurally
  generated across bedroom count (1-6) × plot width × plot depth × 2-3 layout
  orderings — every combination is geometrically distinct (different room
  adjacency, not just resized rectangles) and passes the same
  `GeometryValidator` used everywhere else before being stored, so the
  catalog can't contain a broken template. This is intentionally not framed
  as 1,088 individually hand-designed architectural concepts — it's a large,
  legitimate combinatorial space from one well-tested generator, which is
  what "1000+ templates" can honestly mean at this stage.
- **Drawing sheet PDF export**: the Export modal now has a second tab —
  Drawing Sheet — producing a real scaled architectural sheet (A4/A3/A2,
  landscape): floor plan drawn to scale with walls/rooms/doors/windows,
  overall building width/depth dimension lines, a north arrow, an accurate
  scale bar, and a title block with project name, date, drawn-by, and a
  **blank** ARCON registration line — never fabricated, since this tool must
  never imply a professional certification it hasn't performed. The title
  block also states plainly that the sheet hasn't been reviewed or stamped by
  a registered professional. If the drawing doesn't fit the chosen paper at
  the requested scale, it's automatically loosened to the next standard
  scale to fit rather than clipping — and the sheet says so. **Honest scope
  note**: only overall envelope dimensions are shown, not per-wall/per-room
  dimension strings — that's the natural next increment.
- **Performance fixes**: login and signup were loading the entire ~1.3MB app
  bundle (Three.js, the WebSocket/STOMP client, the whole CAD engine) before
  rendering a simple form. Routes are now lazy-loaded per-page — login's
  chunk dropped to 3.5kB gzipped. Separately, signup was blocking on SMTP for
  however long an unreachable mail server takes to time out (the default
  placeholder Gmail credentials never actually connect); email sending is now
  `@Async` with a 3-second connection timeout, so a slow/misconfigured mail
  server can no longer stall the signup request.
- Also fixed: the Tabler Icons stylesheet was pinned to a version that doesn't
  exist (`2.47.0`), so no icons were rendering anywhere in the app — now
  points at `@latest`. The editor's vertical tool sidebar also got small text
  labels under each icon so tools are identifiable even before that font
  loads (or if an icon name is ever wrong).
- Client-side schema versioning + migration (`schemaMigration.ts`) so old saved
  drawings don't break the editor when the format evolves.
- Client-side geometry validation mirrors the server-side checks so invalid states
  never get autosaved.

**Backend** (`/backend`, Spring Boot 3 + Java 17 + JPA)
- JWT auth (access + refresh tokens, BCrypt passwords), register/login/refresh/
  verify-email/forgot-password/reset-password.
- Project CRUD (including update) with owner + collaborator (OWNER/EDITOR/VIEWER)
  access control enforced in `ProjectService`, not just at the controller layer.
- Collaborator invite/role-update/remove endpoints.
- Drawing snapshot save/load/history/restore endpoints, with a server-side
  `GeometryValidator` that rejects malformed canvas JSON before it's persisted
  (checks wall length, room point count, opening host-wall references, etc.)
  independent of what the client already checked — never trust the client alone.
- **Compliance rule engine**: `ComplianceRuleSet` entity seeded with four rule sets
  (LASDRI, FCTDA, KEBS, NHBRC), and a `ComplianceService` that parses a project's
  latest drawing snapshot and evaluates front/rear/side setbacks, plot coverage %,
  building height, and road-reserve (ROW) clearance against the assigned ruleset.
  **Important caveat**: the seeded setback/coverage/height figures are
  representative planning-standard values for low-rise residential development,
  not verified against each authority's current gazetted regulations — replace
  them with authority-confirmed figures before relying on this for a real
  submission. The engine also assumes the canvas origin is plot-relative with the
  road-facing edge at y=0 (stated explicitly in `ComplianceService`'s Javadoc) —
  accurate for the common case, but a dedicated plot-boundary drawing tool would
  make this exact for irregular or rotated plots.
- Drawing comments (pin comments to an x/y point on the canvas, resolve/delete).
- WebSocket/STOMP endpoint wired up (`/ws`) for the next pass: live cursor/edit
  broadcast for multiplayer editing.
- **AI layout generation**: `AiLayoutService` calls the Anthropic Messages API
  (`claude-sonnet-4-6` by default, configurable) with a prompt describing the
  plot dimensions, building type, and brief, asking for JSON matching the
  frontend's exact CanvasState schema. The response is validated through the
  same `GeometryValidator` used for manual saves before being returned — so a
  malformed AI response never reaches the editor as valid data, it just fails
  with a clear error asking the user to retry. Rate-limited to 5 generations/hour
  via the `aiLayout` resilience4j limiter (returns 429 with a clear message once
  exhausted). Requires `ANTHROPIC_API_KEY` to be set — without it, the endpoint
  returns a 503 explaining AI generation isn't configured, rather than failing
  opaquely.
- **Real-time collaboration backend**: JWT-authenticated WebSocket handshake
  (`JwtHandshakeInterceptor` + `StompHandshakeHandler` — rejects the connection
  outright if the token is missing/invalid, no anonymous WS connections),
  presence tracking (`PresenceRegistry`, cleaned up automatically on disconnect
  via `PresenceDisconnectListener` — not just on explicit "leave"), and a
  `CollabController` broadcasting cursor positions and full-state edits over
  STOMP topics per project. **Known gap, documented in the code**: STOMP
  SUBSCRIBE isn't access-controlled the way REST endpoints are — anyone with a
  valid account who knows a project's UUID can subscribe to its live channels.
  Acceptable for now since project ids are non-guessable, but worth closing
  before this handles anything sensitive. The `state` broadcast channel is
  last-write-wins (see the frontend section above) — `CollabController`'s
  Javadoc spells out exactly what that means and why.
- **BOQ engine**: `BoqService` parses the latest drawing snapshot and computes
  net blockwork area (gross wall area minus door/window openings), plastering
  area, door/window counts, and room floor area — then prices each against
  seeded `MaterialRate` figures (also placeholder market rates, same caveat as
  the compliance figures — see `DataInitializer`). `BoqExcelExportService`
  (Apache POI) and `BoqPdfExportService` (OpenPDF) turn that into styled,
  downloadable files matching a standard categorized-quantity-sheet layout.
- **Template catalog generation**: `ProceduralTemplateGenerator` builds a
  front-living-strip + back-row-of-rooms grid layout for any bedroom count/
  plot size/column-ordering combination, guaranteeing exact shared edges
  between rooms (no gaps, no overlaps) by construction rather than by
  post-hoc collision checking. `TemplateCatalogSeeder` runs the full
  combinatorial sweep at startup (6 bedroom counts × 8 width steps × 8 depth
  steps × 2-3 style orderings = 1,088 valid layouts, verified against Python
  port of the same math before this was wired into Spring — see the
  conversation this was built in), validating every single one through
  `GeometryValidator` before persisting it, and skipping (not fabricating) any
  combination too small to fit its room count. `TemplateSerializer` converts
  the generator's output into the exact CanvasState JSON schema the frontend
  already expects, plus a lightweight schematic SVG thumbnail (colored room
  rectangles, no text — illegible at thumbnail scale anyway) stored per
  template for the gallery grid.
- **Drawing sheet rendering**: `DrawingSheetPdfExportService` uses OpenPDF's
  low-level `PdfContentByte` API to plot the drawing to real-world scale
  (converting meters → mm-on-paper → PDF points based on the chosen scale
  ratio), draws dimension lines/north arrow/scale bar, and composes the
  title block. `DrawingSheetController` exposes it at
  `/api/projects/{id}/drawing-sheet/export.pdf`.
- Global exception handler mapping domain exceptions to proper HTTP codes.
- H2 in-memory DB for local dev, Postgres profile (`application-prod.yml`) for
  production — swap with one env var, no code changes.
- Admin account seeding from `ADMIN_SEED_EMAIL` / `ADMIN_SEED_PASSWORD` env vars
  only — never hardcoded credentials.

## Running it locally

**Backend** (needs Java 17 + Maven):
```
cd backend
mvn spring-boot:run
```
Runs on `http://localhost:8080`. H2 console at `/h2-console` if you want to poke
at the data directly. To enable the AI room planner, set `ANTHROPIC_API_KEY`
before starting it (e.g. `$env:ANTHROPIC_API_KEY="sk-ant-..."` in PowerShell) —
without it, that feature returns a clear "not configured" error instead of
crashing.

**Frontend** (needs Node 18+):
```
cd frontend
npm install
npm run dev
```
Runs on `http://localhost:5173`, proxies `/api` and `/ws` to the backend.

Both have been verified: the frontend passes a full `tsc` type-check and a
production `vite build` with zero errors. The backend was reviewed carefully but
**not** compiled in this environment — Maven Central isn't reachable from this
sandbox's network allowlist, so run `mvn compile` yourself as a first check before
relying on it.

## What's deliberately not built yet

Not included yet, in rough priority order for the next pass:
1. Payments (Paystack/Flutterwave) + subscription plans (enums already exist:
   `SubscriptionPlan`, `SubscriptionStatus`, but no service/controller yet)
2. Per-wall/per-room dimension strings on the drawing sheet (currently only
   overall building envelope width/depth is dimensioned)
3. Foundation/roofing/electrical/plumbing modeling — needed before the BOQ can
   honestly cover those categories, and before templates could include them
4. Material takeoff refinements (currently area/count-based; no wastage
   factors, no material-specific breakdown like cement bags vs blocks)
5. Admin dashboard
6. Known bug: door/window placement silently no-ops if you click too far from
   an existing wall (needs a clearer "not close enough to a wall" affordance)
7. A real merge strategy (CRDT/OT) for simultaneous concurrent editing, if
   this needs to support more than one active editor at a time reliably
8. Polygon-based (non-rectangular) template generation — angled porches and
   corridor-side room access, per the reference floor plans reviewed

## Project structure

```
backend/
  src/main/java/africa/prodesign/
    entity/       User, Project, ProjectCollaborator, DrawingSnapshot, DrawingComment
    repository/   Spring Data JPA repositories for each entity
    security/     JwtService, JwtAuthFilter
    service/      AuthService, ProjectService, CollaboratorService, SnapshotService,
                   CommentService, EmailService
    controller/   AuthController, ProjectController, EditorController
    validation/   GeometryValidator (server-side canvas JSON validation)
    config/       SecurityConfig, WebSocketConfig, GlobalExceptionHandler
    dto/          Request/response records for every endpoint

frontend/
  src/
    engine/       CADEngine (render/hit-test), snapEngine, geometryValidation,
                   schemaMigration, types — all framework-agnostic, testable in isolation
    store/        editorStore (Zustand, undo/redo), authStore
    three/        BuildingScene, SunLight, materials, ThreeDView
    components/editor/  CadCanvas, Toolbar, StatusBar
    pages/        Landing, Login, Signup, Dashboard, Editor
    api/          axios client with auto token-refresh, auth/projects/snapshots calls
```
